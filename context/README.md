# Context ディレクトリ

このディレクトリは、プロジェクト固有の技術知見や注意事項を管理する場所です。

## 使い方

プロジェクトごとに技術知見を記述したファイルを配置します。

```
context/
├── README.md           # このファイル
├── project-a.md       # プロジェクトAの知見
└── project-b.md       # プロジェクトBの知見
```

タスク実行時に、プロジェクトに応じたcontextファイルを参照するように指示します。

## ファイル形式

各contextファイルには以下を含めます：

- プロジェクトの概要
- 使用技術スタック
- 注意事項・禁止事項
- 参考リンク

## テンプレート

```markdown
# {project_id} プロジェクトコンテキスト
最終更新: YYYY-MM-DD

## 基本情報
- **プロジェクトID**: {project_id}
- **正式名称**: {name}
- **パス**: WORKS/{MMDD}{ProjectName}/

## 概要
{プロジェクトの概要を1-2文で}

## 技術スタック
- 言語:
- フレームワーク:
- データベース:

## 重要な決定事項
- {決定1}
- {決定2}

## 注意事項
{プロジェクト固有の注意点}
```

## 更新ルール

- 重要な決定があったら即座に更新
- 日付を必ず更新
- 不要になった情報は削除（シンプルに保つ）
